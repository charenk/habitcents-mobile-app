## 2026-07-04 additions

- **Analytics (P2-3, shipped):** `utils/analytics.ts` is an env-gated PostHog wrapper: complete no-op with zero network unless `EXPO_PUBLIC_POSTHOG_API_KEY` is set (gitignored `.env`, US region). The single sanctioned exception to the no-network rule. Device-verified end to end.
- **Multi-currency (P2-6, shipped):** `utils/currency.ts` (7 currencies, zero-decimal JPY) + `CurrencyContext`/`useCurrency().format`. Money derives from cents at render; the stored `amountDisplay` field was dropped. Detection thresholds scale via `scaleThresholdCents`. All user-facing strings live in `constants/strings.ts` (14 groups); never hardcode UI copy or "$".
- **Corrupt-asset lesson:** the original committed icons were 560-byte invalid PNGs that `file` called valid; they crashed `expo prebuild` with "Unsupported critical chunk type". Diagnose with `xxd`, not `file`. Valid 1024px placeholders are committed; real art is still P0-4.

Native path: CocoaPods 1.16.2 installed (needs `LANG=en_US.UTF-8`), `npx expo run:ios --device <UDID>` (device names contain a curly apostrophe; use the UDID), keep the phone unlocked, and Metro must be running or a Debug build shows a white screen (`curl localhost:8081/status`).
- **Expenses sheet:** `TodayExpensesPanel` snaps at 18% (collapsed default) / 55% / 95%; the add-form scrolls with bottom padding so Save always clears the sheet (stopgap until P2-4).
- **Working mode (locked 2026-07-03): spec-first.** No code until the item's spec .md lands in `../docs/` plus an ADR. `../docs/leak-scan-spec.md` (ADR 0003) re-specifies P2-1b; `docs/design-context/` in THIS repo mirrors the design docs for GitHub-linked design sessions (umbrella `../docs/` stays canonical; re-sync after changes).

## 2026-07-13 additions (domain + web foundation)

- **Canonical domain (ADR 0013, ops repo):** habitcents.com for all URLs, email, and docs. `habitcents.app` is unregistered/dead, never use it. The bundle id `com.habitcents.app` stays: it is a reverse-DNS identifier, immutable in App Store Connect, unrelated to the `.app` TLD.
- **Mobile change this session:** `app/(tabs)/settings.tsx` `PRIVACY_POLICY_URL` is now `https://habitcents.com/privacy` (was the dead `.app`; PR #23). Legal pages `/privacy`, `/terms`, `/support` are live at habitcents.com, so the App Store privacy-URL requirement is now satisfiable.
- **DNS platform lesson:** a domain can stay "linked to another Vercel account" even after you remove it from the old project, because the `vercel-dns.com` nameservers keep the DNS zone under that account. Deterministic fix that needs no access to the old account: at the registrar (Namecheap), switch nameservers Custom DNS -> Namecheap BasicDNS to take DNS control, then add the `_vercel` TXT verification records plus A/CNAME. Verification tokens rotate; always copy the current ones off the Vercel Domains page. Production deploys can still succeed while the PR preview check fails on the Vercel git-author access gate (cosmetic).
- **Concurrent-agent hazard confirmed:** nested git worktrees under an actively-used checkout are fragile. A sibling session's `git clean` wiped one mid-task. For parallel website/mobile work, create worktrees OUTSIDE the main working tree.

## 2026-07-15 additions

- **RevenueCat is mock-first, on main.** `utils/purchases.ts` is env-gated on `EXPO_PUBLIC_REVENUECAT_API_KEY` but `getEntitlement()`/`purchase()`/`restore()` branch on an internal `impl` client that is ONLY set via the test seam. Live wiring needs `react-native-purchases` + a dev build (sequences after Expo login / BET-002) + real Products created in RevenueCat once prices ratify. Paywall `app/paywall.tsx` is a v1 pending design review; prices are PLANNED placeholders.
- **Brand assets refreshed (mark shape UNCHANGED).** Same circular-arrow-plus-checkmark mark; new tile/color treatment. Colors: brand green `#41CE75`, charcoal tile `#232323`, off-white tile `#EEEEEE`. iOS icons are appearance-matched (light = off-white+green, dark = charcoal+white) so neither mode looks wrong; base `icon.png` must be opaque (no alpha) for the App Store. Android adaptive foreground = white mark scaled into the ~66% safe circle over `#232323`, plus a monochrome variant. Splash = green mark on `#F8F8F8`. Canonical SVG masters live in ops `docs/brand/` and mirror to `assets/brand/`. In-app theme green stays `#4CAF50` (intentionally distinct from the brand green; changing it is a deferred design pass).
- **Derivation tooling:** no ImageMagick/rsvg on this machine; use Node + `sharp` + `png-to-ico` (installed in a scratch dir). Recolor the single-path mark SVG by swapping its `fill`; composite onto full-bleed opaque squares for iOS (no rounded corners, iOS masks). Always eyeball generated PNGs (Read the file) before committing since the app can't be built here.
- **Theme-adaptive SVG pattern (website):** the site has no Tailwind `dark:`; theming is pure CSS variables (`--fg` flips white<->near-black via `[data-theme]` + `prefers-color-scheme`). To make one logo work in both themes, inline the SVG (component, not `<img>`) and set the theme-flipping parts to `fill="currentColor"` while keeping the constant brand color hardcoded. `components/Wordmark.tsx` does this: tile + text = currentColor, mark = green.
- **Merge-conflict reality:** long-lived feature branches (BET-004 predated the a11y PR #25) conflict with intervening merges on shared UI files (settings.tsx, PickOneSheet.tsx). Resolve by KEEPING BOTH sides (a11y props + feature wiring), not choosing one. Verify with tsc + jest before completing the merge.
